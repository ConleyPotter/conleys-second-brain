## The Pivot (What Changed and Why)

ACE has a history. Understanding it is essential for accurately reading any document in this wiki.

**The original ACE** (built fall 2025 through February 2026) was an ambitious, fully autonomous creative studio — 8 specialized agents running in a closed learning loop: research → create → test → measure → learn → decide. It was sunsetted. See [[ace-legacy]] for the full picture of what it was.

**The current ACE** emerged from that pivot. Two things drove the change:

1. Feedback revealed that full autonomy in content creation sacrificed quality and strategic intent. A human-in-the-loop approach was essential.
2. The 15-hour weekly bandwidth constraint made a self-running creative studio the wrong abstraction. The right abstraction was **automated output arbitrage** — completing structured, objective tasks through agentic pipelines, where the operator directs the machine rather than managing creative loops.

The pivot did not abandon the technical foundation. Multi-agent orchestration, HITL review gates, observability — all carried forward. What changed was the use case and the commercial model.

**Note on the portfolio:** The portfolio website still describes ACE in terms of the legacy system ("multi-agent learning engine, not a prompt stack... infrastructure with memory"). That copy predates the pivot and has not been updated. It is not accurate to the current ACE. See [[portfolio-website]].

## What ACE Is Now

ACE (Autonomous Content Engine) is Conley Potter's solo-operated freelance arbitrage business. The operator fulfills gigs on Upwork and Fiverr using automated pipelines. Clients receive deliverables that appear to be the result of manual labor; the operator captures the full margin generated by computational efficiency.

ACE currently operates in two modes simultaneously:

**Mode 1 — Human-Collaborative Co-Pilot**
The human operator is the creative lead. Automated agents handle research and drafting. Drafts and workflow outputs are routed to a private channel (Slack or Telegram) via webhooks for asynchronous review, micro-adjustment, and final approval. This model maintains quality and ensures platform compliance.

**Mode 2 — Automated Output Arbitrage**
For structured, objective tasks — B2B lead enrichment, SEO content, marketing workflows — the pipeline runs near-autonomously. These tasks are deterministic enough that human review is lightweight. This is where the margin compression is sharpest.

**The core thesis:** Enterprise clients on Upwork price digital deliverables based on the historical assumption of manual human effort. Those deliverables — B2B data lists, SEO content, marketing assets — can be generated near-instantly via agentic infrastructure. ACE exploits the gap between what clients believe a task costs in human hours and what it actually costs in API calls.

**The hard constraint:** Maximum 15 hours per week. All bandwidth spent directing the machine — never fulfilling manually. See [[conley-potter]] for full operator context.

## The Three-Phase Roadmap

### Phase I — B2B Lead Enrichment (Now)

The immediate service: structured, verified B2B contact lists delivered as CSV files. Chosen because:
- Deliverables are structured and deterministic — no subjective feedback, no scope creep
- Fully automatable (domain → Apollo → Cleanlist → Google Sheet)
- Infinite horizontal scalability at near-zero marginal cost per additional contract

Target niche: manufacturing and industrial firms in Lancaster County, PA. See [[phase-1-lead-enrichment]].

**Phase I objective:** Generate 6–12 months of operating runway before expanding.

### Phase II — Covert Output Arbitrage

Once capital runway is secured, ACE expands into three content vectors:
1. High-volume SEO and local service content (multi-agent: Researcher → Writer → Optimizer → WordPress API)
2. Faceless YouTube script automation (YouTube API + Firecrawl → Gemini/Claude synthesis)
3. Bulk e-commerce asset generation (Shopify API + Flux.1/Leonardo.ai → Shopify upload)

See [[phase-2-content-vectors]].

### Phase III — Overt Infrastructure Deployment

The final transition: instead of using the machine to sell outputs, the operator sells the machine itself.
- High-ticket infrastructure cloning: $5,000–$15,000 per n8n/OpenClaw build
- RevOps retainers: $2,000–$8,000/month for ongoing AI workflow management

See [[phase-3-infrastructure]].

## Emerging Product: The Proactive Ghost

Built alongside the Phase I–III roadmap is a fourth commercial track: **The Proactive Ghost**, a tech-enabled ghostwriting agency-in-a-box built on OpenClaw. Unlike Phase I (selling outputs) or Phase III (selling infrastructure builds), The Proactive Ghost sells a managed service — a private AI ghostwriter that proactively reaches out to clients via messaging apps, converts voice notes into full content ecosystems, and improves its voice accuracy over time through persistent memory files.

Revenue model: $997 one-time setup + $497/month recurring. Near-zero marginal cost per additional client once the infrastructure is built.

This sits closest to Phase III in spirit (selling the machine, not the output), but the recurring MRR structure may make it a parallel track rather than a sequential one. See [[thought-leader-engine]] for full architecture and implementation status.

## Operating Principles

- **Never disclose automation to clients of Covert-labeled projects.** Those clients must believe fulfillment is manual. Never mention n8n, Apify, Apollo, AI models, or pipelines.
- **15-hour constraint is sacred.** Time is the only non-scalable asset. Every system is built to reduce operator involvement.
- **HITL for Upwork compliance.** Upwork's Terms of Service prohibit fully automated proposal submission. The ACE pipeline drafts everything; the operator manually reviews and clicks Submit. See [[client-acquisition]].
- **Covert before overt.** Revenue funds infrastructure. Infrastructure enables scale. Scale enables the transition to selling systems.
