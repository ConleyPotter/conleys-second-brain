# AutoBoost Vault — Skills Development & Team Rollout

## Overview

Conley is preparing to roll out the **AutoBoost Vault** — an internal knowledge management and AI tooling project — to the full BA team. The May 2026 development push focuses on converting personal slash commands into reusable **skills and routines** that non-technical team members can invoke in plain language.

## Key developments

- **Slash commands → skills migration:** Conley has been transitioning his personal Claude Code slash commands into skills — reusable, folder-based bundles of scripts that agents can invoke. This shift enables plain-language access and wider team adoption.
- **Team-wide rollout planned:** The AutoBoost Vault will be announced to the full BA team. A key challenge is making the system friendly enough for non-technical users.
- **Extensibility focus:** Skills and routines allow users to perform more actions without needing to understand the underlying scripts. Files that Conley has been producing manually can now be generated automatically.
- **Hyperagent skills model:** Conley notes that skills in Claude can be structured as folders containing multiple scripts — a model he considers a significant capability upgrade. He has been actively upgrading his skills architecture.

## Implications

- The AutoBoost Vault represents BA's first internal AI tooling rollout beyond Conley's personal use
- Success depends on UX simplification — the system must work for the full 13-person team (see [[ba-team]])
- Skills-based architecture aligns with the broader shift toward agent-native workflows (see [[tech-stack]])
