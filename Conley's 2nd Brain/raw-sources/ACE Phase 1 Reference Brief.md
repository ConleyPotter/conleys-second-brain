**1. What is ACE?**

ACE (Autonomous Content Engine) is a solo-operated freelance arbitrage business built on n8n automation. The operator fulfills Upwork and Fiverr gigs using agentic pipelines — clients receive manual-quality deliverables; the operator captures the full margin generated by computational efficiency. The business is capped at 15 hours/week of active operator time. Automation handles everything else.

  

Core rule: clients must never know that fulfillment is automated. Never disclose the use of n8n, Apify, Apollo, AI models, or any pipeline tooling.

**3. Phase I Objective: B2B Lead Enrichment**

Phase I runs until the operation has generated 6–12 months of OPEX runway. The sole service offering is B2B lead enrichment. This niche was chosen for three reasons:

- Deliverables are structured CSVs — no subjective feedback, no scope creep.
- Fulfillment is fully deterministic (domain → Apollo → Cleanlist → Supabase → Google Sheet).
- Infinite horizontal scalability: accept 10 identical contracts simultaneously at zero marginal cost.

  

**4. Target Micro-Niche**

Initial focus: manufacturing and industrial firms in Lancaster County, PA (operator is based in Palmyra, PA). Rationale:

- Manufacturing = 16%+ of Lancaster County jobs. Dense cluster of targets.
- High CPQL (~$341 avg) makes a $400–600 enriched list an obvious buy for local B2B agencies.
- Key employers/anchors: Dart Container, Armstrong World Industries, Arconic, I-83 corridor industrial parks.
- Decision-maker titles to enrich: Procurement Manager, Plant Director, VP of Operations.

  

Positioning: sell 500-contact verified lists for $400–600 fixed price. COGS = <$30 in API credits. Margin ≈ 95%.

  

**5. Client Acquisition Pipeline (Fully Automated → HITL Submit)**

Upwork ToS prohibits auto-submitting proposals. The pipeline drafts everything; the operator manually clicks Submit. Total per-proposal operator time: ~2 minutes.

  

|   |   |   |
|---|---|---|
|**Step**|**Tool**|**Action**|
|1. Discovery|Apify jupri/upwork|Polls Upwork for lead enrichment jobs; extracts title, budget, description, URL|
|2. Dedup|n8n DataTables (upwork_jobs)|Checks job ID — halts if already seen|
|3. Scoring|Gemini (fast/cheap model)|Scores job 1–10 for fit vs. operator capabilities. Below 8 → discard|
|4. Proposal Draft|Gemini (quality model)|Writes proposal in Spartan tone (see §6). Appends Google Doc artifact link|
|5. HITL Alert|Slack #new-upwork-gigs|Posts full draft + budget + job URL. Operator reviews async|
|6. Manual Submit|Operator|Copies proposal, opens Upwork URL, minor edits if needed, clicks Submit|

  

**6. Spartan Tone Protocol (Proposals)**

All ACE proposals must comply with this protocol. Violations signal AI generation to experienced clients.

  

|   |   |
|---|---|
|**Rule**|**Detail**|
|Under 100 words|Proposals are concise. Every sentence must carry weight.|
|Zero pleasantries|Never use: 'thrilled to apply', 'hope this finds you well', 'excited about this opportunity'|
|Zero robotic transitions|Never use: 'furthermore', 'delve', 'leverage' (as a verb), 'deliverables' (overused)|
|Address pain directly|Extract the specific problem from the job description. Open with the solution, not credentials.|
|Confident, not boastful|State capability matter-of-factly. No superlatives.|
|Never disclose automation|Do not mention n8n, AI, scrapers, Apify, Apollo, or pipelines to clients. Ever.|

  

**7. Fulfillment Pipeline (Lead Enrichment)**

|   |   |   |
|---|---|---|
|**Step**|**Tool**|**Detail**|
|1. Ingest|Google Sheets Trigger / Webhook|Client provides raw CSV or Google Sheet of domains/company names|
|2. Cache Check|Supabase PostgreSQL|Query: was this domain enriched in last 90 days? Hit → free. Miss → call Apollo|
|3. Enrich|Apollo.io API|Returns: email, title, LinkedIn, company size, phone. Batch size 10–20, Wait node 10–15s between batches|
|4. Verify|Cleanlist / Prospeo API|Validates email deliverability. Cascades here only if Apollo returns low-confidence result|
|5. Sanitize|Gemini node|Normalizes company names (removes LLC/GmbH caps/suffixes). Output looks hand-cleaned.|
|6. Write Back|Google Sheets node|Appends enriched rows to formatted output sheet|
|7. Deliver|Slack → Operator|Draft delivery message queued for operator approval. Operator sends link to client.|

  

Rate limiting: Apollo enforces 50–200 calls/min. Use Split In Batches (size: 10–20) + Wait node (10–15s) to stay well below threshold. A 500-row list completes in under 15 minutes unattended.

**11. Critical Operating Rules**

- NEVER disclose automation to clients. They must believe fulfillment is manual.
- NEVER auto-submit proposals. Always review in Slack and submit manually (Upwork ToS).
- NEVER skip the Supabase cache check — every unnecessary Apollo call costs money.
- NEVER use batch sizes above 20 on Apollo without verifying the account's rate limit tier.
- ALWAYS route n8n workflow errors to the dedicated Slack error channel.
- ALWAYS enable 'Continue On Fail' on non-critical enrichment nodes (e.g., LinkedIn URL lookup).
