# Skill: Write Content

## Trigger phrases
"write the brief for [topic]" / "write this brief" / "produce content from brief"

## Process

1. Read the specified brief from `03-BRIEFS/`
2. Read all source notes linked in the brief
3. Read `05-CLAUDE/CLAUDE.md` → "My Voice" section if not already loaded this session
4. Write the full piece in Conley's exact voice
5. Structure: hook → proof → body → closer
6. Every section must add specific value. No filler. If a section doesn't earn its place, cut it.
7. Apply every voice rule from the CLAUDE.md. When in doubt, make it shorter and more direct.

## Voice requirements (summary — full spec in 05-CLAUDE/CLAUDE.md)

- Short direct sentences. Active voice.
- Specific details over abstractions. Real numbers always.
- Honest about what's broken. No lessons packaging.
- Intensity present but not loud.
- No banned phrases (see Hard Rules in CLAUDE.md).
- No emojis in body text.

The output must be indistinguishable from content Conley wrote himself.

## Output

Save draft in `03-BRIEFS/` next to the source brief.
Example: if brief is `2026-04-25-pipeline-failure-at-2am.md`, draft is `2026-04-25-pipeline-failure-at-2am-draft.md`
Tag it `#written`

## Self-check before saving

Read the draft aloud (or simulate doing so). Flag any line that:
- Sounds like a marketing pitch
- Uses a banned phrase
- Makes a claim without specific proof
- Could embarrass BA or AutoBoost

Rewrite flagged lines before saving.
